Cloud Computing Learning Path

Master cloud computing from fundamentals to advanced multi-cloud architectures

Module 1: Cloud Fundamentals

What Is Cloud Computing?

Cloud computing is the on-demand delivery of computing resources—servers, storage, databases, networking, software, and analytics—over the internet. Instead of buying and maintaining physical data centers and servers, you rent access to technology services from a cloud provider on a pay-as-you-go basis.

This model eliminates the upfront capital expense of purchasing hardware, reduces the ongoing costs of running data centers, and lets you scale resources up or down in minutes rather than weeks.

Service Models

  • IaaS (Infrastructure as a Service): Provides virtualized computing resources over the internet. You manage the operating system, applications, and data while the provider manages the hardware, networking, and virtualization. Examples: AWS EC2, Azure Virtual Machines, Google Compute Engine.
  • PaaS (Platform as a Service): Provides a complete platform for developing, running, and managing applications without dealing with the underlying infrastructure. Examples: AWS Elastic Beanstalk, Azure App Service, Google App Engine.
  • SaaS (Software as a Service): Delivers fully functional software applications over the internet on a subscription basis. Users access the software through a web browser. Examples: Gmail, Salesforce, Microsoft 365, Slack.

Deployment Models

  • Public Cloud: Resources are owned and operated by a third-party cloud provider and shared across multiple organizations. Most cost-effective for common workloads with standard compliance requirements.
  • Private Cloud: Cloud infrastructure used exclusively by a single organization. Offers greater control, customization, and security but comes at a higher cost.
  • Hybrid Cloud: Combines public and private clouds, allowing data and applications to move between them. Provides flexibility for sensitive workloads while leveraging the public cloud's scale.
  • Multi-Cloud: Using services from multiple cloud providers simultaneously to avoid vendor lock-in and leverage the best-of-breed services from each provider.

Key Benefits

  • Scalability: Scale resources up or down based on demand within minutes.
  • Cost Efficiency: Pay only for what you use with no upfront capital expenditure.
  • Global Reach: Deploy applications in data centers around the world in minutes.
  • Reliability: Built-in redundancy, automated backups, and disaster recovery capabilities.
  • Speed: Provision new resources in minutes instead of the weeks it takes to procure physical hardware.
  • Security: Major cloud providers invest billions annually in security and hold numerous compliance certifications.

Module 2: AWS Deep Dive

AWS Overview

Amazon Web Services is the world's most comprehensive cloud platform, offering over 200 fully featured services from data centers across 31 geographic regions. AWS serves millions of customers including startups, large enterprises, and government organizations. It holds the largest market share in the cloud industry.

EC2 (Elastic Compute Cloud)

EC2 provides resizable virtual servers in the cloud. You can launch instances with different configurations of CPU, memory, storage, and networking capacity. EC2 instances are the backbone of most AWS architectures.

  • Instance families: General Purpose (t3, m6i), Compute Optimized (c6i), Memory Optimized (r6i), Storage Optimized (i3), GPU (p4d)
  • Pricing models: On-Demand (pay per second), Reserved Instances (1-3 year commitment, up to 72% savings), Spot Instances (up to 90% savings for interruptible workloads)
  • Auto Scaling Groups automatically adjust the number of instances based on demand
  • Elastic Load Balancers distribute incoming traffic across multiple instances

S3 (Simple Storage Service)

S3 is an object storage service designed for 99.999999999% (11 nines) of data durability. It stores data as objects within buckets and is used for backups, static website hosting, data lakes, content distribution, and application data storage.

  • Storage classes: S3 Standard, Intelligent-Tiering, Standard-IA, One Zone-IA, Glacier Instant Retrieval, Glacier Flexible Retrieval, Glacier Deep Archive
  • Features: versioning, lifecycle policies, cross-region replication, event notifications
  • Security: bucket policies, ACLs, server-side encryption (SSE-S3, SSE-KMS, SSE-C), block public access settings

VPC (Virtual Private Cloud)

VPC lets you create a logically isolated virtual network in the AWS cloud. You have complete control over your network configuration including IP address ranges, subnets, route tables, and gateways. VPC is fundamental to securing your AWS resources.

  • Subnets: public subnets (internet-accessible) and private subnets (isolated) across multiple Availability Zones
  • Internet Gateways for public internet access, NAT Gateways for outbound internet access from private subnets
  • Security Groups (stateful instance-level firewall) and Network ACLs (stateless subnet-level firewall)
  • VPC Peering, Transit Gateway, and VPN for connecting multiple networks

IAM (Identity and Access Management)

IAM controls who can access your AWS resources and what actions they can perform. It is the cornerstone of AWS security and should be the first service you learn thoroughly.

  • Users, Groups, Roles, and Policies form the core of IAM
  • Policies are JSON documents specifying Effect (Allow/Deny), Actions, and Resources
  • Always follow the Principle of Least Privilege: grant only the permissions needed
  • Enable Multi-Factor Authentication (MFA) on all accounts, especially the root account
  • Use IAM Roles for applications and services instead of embedding access keys

Lambda

AWS Lambda is a serverless compute service that runs your code in response to events without requiring you to provision or manage servers. You pay only for the compute time consumed, measured in milliseconds. Lambda supports Python, Node.js, Java, Go, .NET, and Ruby.

  • Triggers include API Gateway, S3 events, DynamoDB Streams, SQS, CloudWatch Events, and more
  • Maximum execution time of 15 minutes per invocation
  • Scales automatically from zero to thousands of concurrent executions

RDS (Relational Database Service)

RDS makes it easy to set up, operate, and scale relational databases in the cloud. It handles routine database tasks such as provisioning, patching, backup, recovery, and scaling. RDS supports MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Amazon Aurora.

  • Multi-AZ deployments for high availability with automatic failover
  • Read Replicas for read-heavy workloads and improved performance
  • Automated backups with point-in-time recovery up to 35 days

Other Essential AWS Services

  • Route 53: Scalable DNS and domain registration service
  • CloudFront: Content delivery network (CDN) for fast global content distribution
  • CloudWatch: Monitoring and observability for AWS resources and applications
  • CloudTrail: Records API calls and account activity for auditing and compliance
  • SNS/SQS: Messaging services for decoupling application components
  • DynamoDB: Fully managed NoSQL database with single-digit millisecond performance
  • ECS/EKS: Container orchestration services (ECS for Docker, EKS for Kubernetes)

Module 3: Azure Deep Dive

Azure Overview

Microsoft Azure is the second-largest cloud platform, operating in over 60 regions worldwide. Azure integrates seamlessly with Microsoft's enterprise ecosystem including Active Directory, Office 365, and Windows Server, making it the natural choice for organizations already invested in Microsoft technologies.

Azure Virtual Machines

Azure VMs provide on-demand, scalable computing resources supporting both Windows and Linux. Azure offers a wide range of VM sizes optimized for different workloads including general purpose, compute-intensive, memory-intensive, and GPU-enabled configurations.

  • VM Scale Sets for automatic scaling of identical VMs
  • Availability Sets and Availability Zones for high availability
  • Azure Spot VMs for cost savings on interruptible workloads (up to 90% discount)

Azure Blob Storage

Blob Storage is Azure's object storage solution optimized for storing massive amounts of unstructured data such as text, binary data, images, and videos. It offers three access tiers to help you store data cost-effectively based on access patterns.

  • Access tiers: Hot (frequently accessed), Cool (infrequently accessed, stored for at least 30 days), Archive (rarely accessed, stored for at least 180 days)
  • Lifecycle management policies to automatically transition data between tiers
  • Geo-redundant storage (GRS) for cross-region data replication

Azure Virtual Network (VNet)

VNet is the fundamental building block for private networks in Azure. It enables Azure resources to securely communicate with each other, the internet, and on-premises networks. VNets are scoped to a single region but can be connected across regions using VNet Peering.

  • Subnets for network segmentation within a VNet
  • Network Security Groups (NSGs) for filtering inbound and outbound traffic
  • Azure VPN Gateway and ExpressRoute for hybrid connectivity
  • Azure Firewall and Azure DDoS Protection for advanced security

Azure Active Directory (Azure AD / Entra ID)

Azure AD (now Microsoft Entra ID) is a cloud-based identity and access management service. It handles authentication and authorization for Azure resources, Microsoft 365, and thousands of third-party SaaS applications. It is fundamentally different from on-premises Active Directory.

  • Single Sign-On (SSO), Multi-Factor Authentication (MFA), and Conditional Access
  • Role-Based Access Control (RBAC) for fine-grained Azure resource permissions
  • Azure AD B2C for customer identity management

Azure Functions

Azure Functions is a serverless compute service for running event-triggered code without managing infrastructure. It supports C#, JavaScript, Python, Java, and PowerShell. Functions integrate natively with other Azure services and can be triggered by HTTP requests, timers, queues, and database changes.

Other Essential Azure Services

  • Azure App Service: Fully managed platform for building web apps and APIs
  • Azure SQL Database: Fully managed relational database service
  • Azure Kubernetes Service (AKS): Managed Kubernetes for container orchestration
  • Azure DevOps: Suite of development tools for CI/CD pipelines
  • Azure Monitor: Full-stack monitoring for applications and infrastructure
  • Azure Key Vault: Centralized secrets, keys, and certificate management

Module 4: GCP Deep Dive

GCP Overview

Google Cloud Platform leverages the same infrastructure that powers Google Search, YouTube, and Gmail. GCP is particularly strong in data analytics, machine learning, and Kubernetes (which Google originally created). It operates in over 35 regions worldwide and is the third-largest cloud provider.

Compute Engine

Compute Engine delivers virtual machines running in Google's global data centers. It offers predefined and custom machine types, preemptible VMs for batch workloads at significant discounts, and live migration that moves running VMs between hosts with zero downtime during maintenance.

  • Machine families: General-purpose (E2, N2), Compute-optimized (C2), Memory-optimized (M2), Accelerator-optimized (A2)
  • Preemptible VMs and Spot VMs for up to 91% cost savings
  • Managed Instance Groups with autoscaling and autohealing

Cloud Storage

Cloud Storage is a unified object storage service with a single API and strong consistency across all storage classes. It provides four storage classes to match different access patterns and cost requirements.

  • Storage classes: Standard, Nearline (accessed less than once per month), Coldline (less than once per quarter), Archive (less than once per year)
  • Object Lifecycle Management for automatic class transitions and deletion
  • Dual-region and multi-region options for high availability

Google Kubernetes Engine (GKE)

GKE is a managed Kubernetes service that provides a production-ready environment for containerized applications. Since Google created Kubernetes, GKE offers the most advanced and tightly integrated Kubernetes experience among all cloud providers, with features like Autopilot mode for fully managed clusters.

  • GKE Standard for full cluster control, GKE Autopilot for hands-off management
  • Integrated logging and monitoring with Cloud Operations suite
  • Workload Identity for secure access to GCP services from pods

BigQuery

BigQuery is a fully managed, serverless data warehouse that enables super-fast SQL queries over petabytes of data. It separates storage and compute, meaning you can scale each independently. BigQuery is one of GCP's most compelling and differentiated services.

  • Serverless architecture with no infrastructure to manage
  • Built-in machine learning with BigQuery ML
  • On-demand and flat-rate pricing options
  • Real-time analytics with streaming inserts

Other Essential GCP Services

  • Cloud Functions: Serverless event-driven compute (similar to Lambda)
  • Cloud Run: Fully managed serverless containers
  • Cloud SQL: Managed MySQL, PostgreSQL, and SQL Server
  • VPC: Virtual Private Cloud networking with global scope
  • Cloud IAM: Fine-grained identity and access management
  • Pub/Sub: Real-time messaging service for event-driven architectures
  • Cloud Build: Serverless CI/CD platform

Module 5: Advanced Cloud Concepts

Multi-Cloud Strategy

A multi-cloud strategy involves using services from two or more cloud providers to avoid vendor lock-in, optimize costs, meet regulatory requirements, and leverage the strongest services from each provider. Tools like Terraform, Pulumi, and Crossplane help manage infrastructure across multiple clouds with a unified workflow.

  • Avoid vendor lock-in by abstracting cloud-specific APIs where possible
  • Use Kubernetes as a portable platform layer across clouds
  • Implement consistent identity, security, and networking policies across providers

Cloud Security Best Practices

Security in the cloud follows the Shared Responsibility Model: the cloud provider secures the infrastructure, while you are responsible for securing your data, applications, and configurations. Understanding this boundary is critical.

  • Apply the Principle of Least Privilege to all IAM policies
  • Encrypt data at rest and in transit using provider-managed or customer-managed keys
  • Enable comprehensive logging and monitoring (CloudTrail, Azure Monitor, Cloud Audit Logs)
  • Implement network segmentation with VPCs, security groups, and private endpoints
  • Regularly audit configurations with tools like AWS Config, Azure Policy, or Security Command Center
  • Use Cloud Security Posture Management (CSPM) tools to detect misconfigurations

Cost Optimization

  • Right-size instances by analyzing actual CPU, memory, and network utilization
  • Use Reserved Instances, Savings Plans, or Committed Use Discounts for predictable workloads
  • Leverage Spot or Preemptible instances for fault-tolerant and batch processing workloads
  • Implement auto-scaling to match capacity with real-time demand
  • Use appropriate storage tiers and lifecycle policies to move cold data to cheaper classes
  • Set up billing alerts, budgets, and cost anomaly detection
  • Review and eliminate unused resources: unattached EBS volumes, idle load balancers, old snapshots

Well-Architected Framework

All three major cloud providers offer a Well-Architected Framework providing best practices for building secure, reliable, efficient, and cost-effective systems. While the exact terminology varies, they share five common pillars:

  • Operational Excellence: Run and monitor systems to deliver business value and continuously improve processes
  • Security: Protect information, systems, and assets through risk assessment and mitigation strategies
  • Reliability: Ensure workloads perform their intended function correctly and consistently, and recover quickly from failures
  • Performance Efficiency: Use computing resources efficiently to meet system requirements and maintain efficiency as demand changes
  • Cost Optimization: Avoid unnecessary costs by understanding spending patterns and controlling fund allocation

Cloud Architecture Patterns

  • Microservices: Decompose applications into small, independently deployable services that communicate via APIs
  • Event-Driven Architecture: Use events to trigger and communicate between decoupled services for real-time processing
  • Serverless: Build applications using managed services and functions, eliminating server management entirely
  • Containerization: Package applications with their dependencies into containers for
